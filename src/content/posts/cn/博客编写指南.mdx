# 博客编写指南

## 1. 前言

本文档旨在帮助团队成员规范、高效地在本项目中编写博客文章，提升内容质量与一致性。

## 2. 文件命名与存放

- 中文博客存放于：`src/content/posts/cn/`
- 英文博客存放于：`src/content/posts/en/`
- 文件名建议使用英文、数字、短横线，避免空格和特殊字符。
  
示例：`ai-application-trends.mdx`、`2024-new-features.mdx`

## 3. Frontmatter 规范

每篇博客需在顶部添加 Frontmatter，示例：

```mdx
---
title: "示例标题"
description: "一句话描述文章内容。"
pubDate: 2024-01-01
image:
  source: "@assets/images/blog/sample.jpg"
  alt: "封面图描述"
tags: ["标签1", "标签2"]
---
```

## 4. Markdown/MDX 语法建议

- 支持标准 Markdown 语法（标题、列表、表格、引用、代码块等）。
- 支持 MDX 组件（需先 import）。
- 推荐使用语义化标签，避免滥用样式。
- 代码块示例：

```js
console.log('Hello, world!');
```

- 引用组件示例：

```mdx
import CustomAlert from '../../components/CustomAlert.astro'

<CustomAlert type="info">这是一个信息提示。</CustomAlert>
```

## 5. 多语言支持

- 建议中英文分开撰写，分别存放于 `cn/` 和 `en/` 目录。
- 标签、描述等 Frontmatter 字段请用对应语言。

## 6. 媒体资源引用

- 图片建议放在 `src/assets/images/blog/` 目录。
- 引用图片时使用 `@assets` 别名：

```mdx
![示例图片](@assets/images/blog/sample.jpg "图片标题")
```

- 视频、音频等媒体建议放在 `public/videos/`、`public/audios/` 目录。

## 7. 数学公式与脚注

- 数学公式支持行内 `$...$` 和块级 `$$...$$` 语法。
- 脚注示例：

```mdx
这是一个带脚注的句子。[^1]

[^1]: 这里是脚注内容。
```

## 8. 常见问题与排查

- 图片不显示：检查路径、文件名、别名配置。
- 组件报错：确认已 import 并路径正确。
- 公式未渲染：确认已配置 remark-math/rehype-katex。
